Why things get messy

  • Different editors which introduce confusion like tabs vs spaces etc
  • Use of tools like jsbeautifier.org
  • Use of git version control after beautifying elsewhere
  • Importing packages which are never used in that module
  • Modules become large creating more merge conflicts in Git

Solution

Use Eslint & Prettier.

package.json

Plugins: yarn optional

Eslint

Website: https://eslint.org/

.eslintrc

Plugins : ESLint by dbaeumer.vscode-eslint

Prettier

Website: https://prettier.io/

.prettierrc

Plugins esbenp.prettier-vscode

Next steps

  • Install Plugins
  • Open settings in VSCode and add
  • Reload VSCode
  • Run the tasks which run linting and prettifying from cli. You may add include & exclude directories. By default node_modules is excluded.
  • Request team to work on prelinted code.
  • Check if prettier is working fine on all machines
  • Update and save code to see prettier doing the magic for you.
  • eslint will validate/lint on type of code.

There you go. It should work now.

Tools referenced

  • VS Code
  • Yarn
  • Eslint
  • Prettier
  • VS code extensions

Now your editor facilitates better code maintainability and Scaling.

For instance, the linting setup helped me refactor 5000 LOC of nodeJS code in 1 day. The unused packages, missing dependencies were clearly informed to the developer. I have to satisfy missing dependencies and remove the rest. Errors like modified const were highlighted.
