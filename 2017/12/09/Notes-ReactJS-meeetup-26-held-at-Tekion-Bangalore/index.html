Notes: ReactJS meetup held at Tekion, Bangalore

Background

ReactJS has a vibrant community in Bangalore. They conduct meetups via meetup.com. I attended their meeting #29. This was the first React meetup I had attended.

Highlights

  • Tekion hosted the meetup. Its a company focussed on bringing IOT closer to automobile retail outlets. It was founded by a person named Jay Vijayan who was working with Elon Musk at Tesla earlier.
  • Venue was pleasant and filled with attendees
  • Speakers were awesome and showed adequate preparation. The topics covered were.
    • Application with React & Apollo
    • Redux middleware
    • Push notifications and firebase
    • Modules and packaging for Node packages
    • React Router V4
    • Immutable.js and immutable concepts
  • Speakers has got some freebies like Stickers. I got react, electron and VS Code sticker.

Key points

  • GraphQL is a spec like REST
  • Apollo is a lib. Relay is alternate from Facebook
  • GraphQL can replace Redux for state management
  • Redux middleware can be used for awesome analytics, getting to know the prev state, next state and action.
  • Push notifications get complicated not because of user permission but when the number of clients and their states are combined.
  • React router V4 is quite modern in many ways. Loved the server side solution offered. Learn react router and the same can be used for React on the server side, React native etc.
  • It was amazing to know how immutable.js works under the hood. Trio was the new term I heard.
  • Never thought packaging for NPM was so hard, was intrigued by the module format and their use cases.
  • When you export default {baz, foo}. We cannot tree shake if foo was not used else where. Instead write export {baz, foo}

Tech terms that were used

  • GraphQL, Apollo & Relay
  • React, Redux & Router
  • Immutable Object.assign etc
  • JS module systems
  • Webpack, Babel & Rollup
  • Tree shaking
  • Push notification

Tools used by speakers

  • VS CODE
  • Redux devtools
  • Apollo devtools
  • Firebase & service workers
  • Slides.com
  • Go lang (Was recommended)
