Tech setup of my blog

Creating your first blog is daunting because of the decisions you have to make. There are multiple options, each with its own set of pros and cons. Here is what went down behind the scenes of my blog.

Options out there related to authoring blogs

There are definitely plenty of options we have when we were to publish a blog. Some of them are

  • Create and publish a Medium Blog
  • Create a wordpress blog
  • Create a blog using static site generator

Medium blogs are good but I want the hacker style, fully customizable with no follow buttons etc. Wordpress requires some php understanding to customize it, which I dont have.

Static sites are cool because there is no known limitation on this platform. You author in markdown & publish on github. No banner ads or components from the platform. If needed, you can add your domain name.

What I chose?

I chose Static Site Generators. I got a hang of Jekyll blog however the ruby gems and related jargon turned me off. I found about Hexo site generator from a tech course on pluralsight. The course discusses about

  • Hexo
  • Docpad

Hexo was simple and easy to learn. So I went ahead with building static websites with Hexo.

You will be writing in markdown and it compiles to html.

Hosting platform

There are 2 ways you can host your blog.

  • Hosting on your own hosting platform
  • Hosting for free on github

Github gives you github pages. Github pages is free & comes with https pre installed. Hexo comes with tools to deploy the static site directly to github pages.
