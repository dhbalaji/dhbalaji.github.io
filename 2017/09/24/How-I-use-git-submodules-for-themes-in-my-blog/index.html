How I use git submodules for themes in my blog

24-09-2017
#BlogStories #Git
Technology

Using Git Submodules for maintaining the themes in HEXO site.

What is a submodule?

Git Submodule is analogous to node packages in JavaScript. The difference being, we are not interested in updating or modifying the node packages. In case of git submodules, the developer is interested in updating the git submodules.

  • Node packages -> Read only, publicly available.
  • Git submodules -> Read & write, private within an organization.

Let me answer few FAQs on git submodules

  • How will the commit history be like? You cannot see the commits of the submodules. It prevents your history looks like spagetti.

  • How can I create a submodule inside a git repo? Its not a tedious process. Docs here

  • What are the advantages of sub modules?

    • Cleaner history
    • Smaller repos
    • One repo can depend on multiple sub modules

  • Will sub module code be referenced inside main repo? The reference to the folder exists. There will be an another file .gitmodules added to the codebase. The code will not be present as part of the main repo. You need to include .gitmodules along with the repo.

  • Why there are not many projects using git modules? I am not sure, there are some opinions and better options like subtrees.

  • Can submodule depend on other submodule? Yes, you need to use --recursive to perform actions on all the sub modules.

Why submodule for themes?

  • Theme files must be loosely coupled to the hexo blog codebase.
  • Loose coupling helps us to place the theme in a seperate repo so others can reuse the theme.
  • Other developers can work on the theme, they need not have access to my codebase which has posts.
  • The history of master repo will be clean. It will not contain silly history of commits like “Fixed Responsive bugs”. Instead the history will have relevant entries only.

How does your setup differ for a git repo without submodules?

Use git clone --recursive URL

You can sync from git submodule sync to update. Refer docs for more commands.

Check status via git submodule status

Init via git submodule init and git submodule deinit

For normal users, go inside the folders to run your normal git commands
